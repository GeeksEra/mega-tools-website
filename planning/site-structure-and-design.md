## DESIGN SYSTEM

### Color Palette
Primary Brand: #6C63FF (Purple — modern, tech, trustworthy)
Secondary: #00D4FF (Electric blue — AI/tech associations)
Background Dark: #0A0A1A (Deep dark navy)
Background Light: #F8F9FF (Near-white with blue tint)
Surface Dark: #12122A (Card backgrounds in dark mode)
Surface Light: #FFFFFF (Card backgrounds in light mode)
Text Primary Dark: #F0F0FF (Near-white text)
Text Primary Light: #1A1A2E (Near-black text)
Accent CTA: #FF6B35 (Orange-red for buttons — high CTR)
Success: #00C896 (Green)
Warning: #FFB347 (Amber)
Error: #FF4757 (Red)
Border Dark: rgba(255,255,255,0.1)
Border Light: rgba(0,0,0,0.08)

### Typography
Headings: "Space Grotesk", sans-serif (modern, geometric)
Body: "Inter", sans-serif (clean, readable at all sizes)
Code: "JetBrains Mono", monospace (developer tools)
Fallbacks: system-ui, -apple-system, sans-serif

Scale:
- H1: 48px / 3rem (homepage hero)
- H2: 36px / 2.25rem (section titles)
- H3: 24px / 1.5rem (tool titles)
- H4: 20px / 1.25rem (subsections)
- Body: 16px / 1rem
- Small: 14px / 0.875rem
- Tiny: 12px / 0.75rem

### Spacing System (8px base)
xs: 4px | sm: 8px | md: 16px | lg: 24px | xl: 32px | 2xl: 48px | 3xl: 64px

### Border Radius
sm: 6px | md: 12px | lg: 16px | xl: 24px | full: 9999px

## PAGE TEMPLATES

### Homepage Layout
1. Navbar: Logo | Category Nav | Search | Dark Mode Toggle | Pro Button
2. Hero: Headline "1000+ Free Online Tools" + search bar (Google-style)
3. Category Grid: 12-16 cards, icon + name + tool count + popular tool links
4. Trending Tools: 6 cards for trending/new tools
5. AI Tools Section: Featured AI tools with descriptions
6. Stats Bar: "500+ Tools | 10M+ Uses | 150+ Countries | 100% Free"
7. Popular Tools Grid: Top 24 tools by traffic
8. How It Works: 3-step visual (Select tool, Use it, Done)
9. User Testimonials
10. Footer: Category links, Popular tools, About, Blog, Privacy, Terms

### Category Page Layout
1. Category hero: Name + description + tool count
2. Search/filter bar (filter by type within category)
3. Tools grid (12-24 tools per page with name, icon, description, use button)
4. Sidebar: Related categories
5. SEO content block (300+ words about the category for Google)
6. FAQ section (5-8 questions)

### Individual Tool Page Layout
1. Breadcrumb: Home > Category > Tool Name
2. Tool title (H1 = keyword)
3. Tool description (1-2 sentences)
4. THE TOOL ITSELF (above the fold on mobile)
5. Ad placement (below tool, not inside it)
6. How to use section (numbered steps)
7. Tool features list
8. FAQ (5-10 questions with schema markup)
9. Related tools grid (6-8 similar tools)
10. SEO content (300+ words for Google ranking)
11. Footer

### Tool UI Design Principles
- Input area: Large, clear, with placeholder text
- Output area: Clear, with copy button
- Action buttons: High contrast, large enough to tap on mobile
- Loading states: Smooth animations, progress indicators
- Error states: Clear error messages with suggested fixes
- Empty states: Helpful prompts to guide user
- Success states: Clear confirmation

## TECH STACK

### Frontend
- Framework: Next.js 14+ (App Router, React Server Components)
- Styling: Tailwind CSS v4 (utility-first, fast)
- UI Components: shadcn/ui (accessible, customizable)
- Icons: Lucide React (consistent, lightweight)
- Animations: Framer Motion (smooth, performant)
- State: Zustand (lightweight global state)
- Forms: React Hook Form + Zod (validation)

### Backend / API
- Runtime: Node.js / Next.js API Routes
- Database: PostgreSQL via Supabase (users, usage tracking)
- Auth: Clerk or NextAuth.js (social + email auth for Pro users)
- File Storage: Cloudflare R2 (S3-compatible, cheap bandwidth)
- AI Integration: OpenAI API (GPT-4o for AI tools)
- PDF Processing: pdf-lib, pdfjs (client-side for free, server-side for Pro)
- Image Processing: Sharp.js (server-side compression, conversion)

### Infrastructure
- Hosting: Vercel (Frontend + API routes)
- CDN: Cloudflare (DDoS protection, global caching)
- Monitoring: Vercel Analytics + Sentry (error tracking)
- Payments: Stripe (subscriptions)
- Email: Resend or Postmark (transactional)
- Search: Algolia or MeiliSearch (tool search)

### Performance Targets
- PageSpeed Score: 90+ mobile, 95+ desktop
- LCP: Under 2.5s
- INP: Under 200ms
- CLS: Under 0.1
- TTFB: Under 800ms (Vercel Edge achieves 200-300ms globally)
- Bundle size: Under 100KB initial JS

## MOBILE-FIRST DESIGN RULES

1. Tool must be FULLY functional on mobile (no desktop-only tools)
2. Touch targets: minimum 44x44px for all buttons
3. Input areas: large enough to type on mobile keyboard
4. Output areas: scrollable, with large copy button
5. No horizontal scrolling
6. Ads: never overlap with tool functionality on mobile
7. Font size: minimum 16px for inputs (prevents iOS zoom)
8. Viewport meta tag: width=device-width, initial-scale=1

### Breakpoints
Mobile: 0-767px (default, mobile-first)
Tablet: 768-1023px
Desktop: 1024-1279px
Wide: 1280px+

---

## DARK MODE IMPLEMENTATION

- Default: Dark mode (tech users prefer dark, better for eyes)
- Toggle: System preference detection + manual toggle
- Store preference: localStorage
- CSS variables for instant switching (no flash)
- All images must work in both modes
- Chart/graph colors: distinct in both modes

---

## ACCESSIBILITY REQUIREMENTS (WCAG 2.1 AA)

- All images: alt text required
- All interactive elements: keyboard navigable
- Color contrast: minimum 4.5:1 ratio
- Focus visible: clear focus rings on all interactive elements
- Screen reader: ARIA labels on all icon-only buttons
- Form inputs: associated labels
- Error messages: connected to form fields via aria-describedby
